Principles

Design Foundations

Four ideas shape how Satori looks and feels, from the marketing website to the analytics platform.

○

Breathing Space

When something feels cramped, add more space. Generous whitespace lets the content speak. Hide sections until they have content.

◑

Typography Does the Work

Three fonts, three roles. Poppins for gravitas, Manrope for readability, Source Code Pro for data. No decorative fonts, no trend-chasing.

◉

Single Accent

One blue for links, active states, and hover effects. No competing colors. Navy for text, never pure black. Calm and focused.

✕

When in Doubt, Remove

Every element must earn its place. If something feels decorative, delete it. If a trend doesn't serve understanding, skip it. Restraint is the feature.

Components

UI Elements

Links preferred over buttons. Outline style, not elevation. Subtle hover states.

Component Philosophy

Prefer link arrows over buttons for most CTAs. Reserve buttons for form submissions or destructive actions. Cards use border not box-shadow — on hover, the border turns accent blue. Inputs are minimal: bottom-border only, no box.

Guidelines

Do's and Don'ts

Do
✓  Generous whitespace — let content breathe
✓  Typography does the heavy lifting
✓  Hero gradient for key pages
✓  Progressive disclosure — show less by default
✓  Centered layouts for focused content
✓  Subtle hover states (border color change)
✓  Friendly, inviting copy in empty states
✓  Monospace for data, dates, technical info
✓  Overlays use useState, not routes
✓  "Continue" not "Next" — forward momentum
Don't
✕  Heavy borders and shadows
✕  Multiple accent colors
✕  Pill-shaped buttons (rounded-full on rectangles)
✕  Crowded layouts with lots of sections
✕  Empty placeholder sections — hide until content
✕  ALL CAPS for headings (only for tags)
✕  Buttons when links would do
✕  Generic "Submit" / "Cancel" — be specific
✕  Page routes for modals/wizards
✕  Skeleton everything — only dynamic data
